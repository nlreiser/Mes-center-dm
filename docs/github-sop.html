7 Github SOP

7.1 Adding a Chapter to this site.

7.1.1 Adding the RMarkdown File Locally

  1. In the local Data Request folder for the Mesulam Center Data Management Practices site, open the .Rproj file. Create a new r Markdown file and save it in the following format: chapter number-title.Rmd (ex: 06-github-sop.Rmd)

  2. Add Headings: A single # is used to title the chapter page. (Github SOP at the top of the page) ## Double ## is used for a Heading and Section Title in the Table of Contents on the right hand side. ### Triple ### is used for a Subheading and Sub-section Title in the Table of Contents on the right hand side. Note: Github and Bookdown automatically numbers the Table of contents based on the saved name of the R Markdown file.

  3. Add desired body of text.

  4. Save the Rmarkdown file. Close the file and open the project file in RStudio.

7.1.2 Render the Project Locally

  1. To render the book, run the following command: bookdown::render_book(“index.Rmd”)
  2. To view the book in html format, run the following command: browseURL(“docs/index.html”)
  3. Confirm all changes were successfully rendered.

7.1.3 Commit the Changes

  1. Open GitHub Desktop. Confirm you are on the correct Repository.
  2. Click the ‘Fetch Origin’ button to view local changes that are to be committed.
  3. Review changes, and if everything looks correct, add a summary and click ‘Commit to main’.
  4. Open github.com in a new browser window and go to the desired Repository.
  5. Click on the ‘Actions’ tab at the top of the page.
  6. A new workflow will run, titled ‘pages build and deployment’. To view progress, click on the action. You will see a pipeline with the build’s progress. If there are no errors, all three steps will have green checkboxes.
  7. When workflow is complete, click the link under ‘deploy’ to view the changes made to the site.
